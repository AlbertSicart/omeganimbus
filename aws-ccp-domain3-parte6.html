01 · Los 3 tipos de almacenamiento en la nube

Objetos · Archivos · Bloques

TIPO 1
Almacenamiento de objetos

Almacena datos como objetos en buckets. Sin estructura de carpetas. Escala ilimitada. Accesible vía HTTP/S.

Amazon S3, S3 Glacier
ilimitado · global · objetos · buckets
TIPO 2
Almacenamiento de archivos

Sistema de archivos compartido montable por múltiples instancias simultáneamente. Ideal para repositorios y directorios de usuario.

Amazon EFS, Amazon FSx
compartido · NFS · SMB · múltiples instancias
TIPO 3
Almacenamiento en bloques

Volúmenes dedicados por instancia. Baja latencia. Para SO, BD y apps de alto rendimiento. Se monta como disco.

Amazon EBS, Instance Store
bloques · dedicado · baja latencia · volumen
02 · Amazon S3 — Clases de almacenamiento

La clase correcta para cada caso de uso

Clase Acceso Recuperación Coste Caso de uso
S3 Standard Frecuente Inmediata Alto Contenido web activo, apps, datos de producción.
S3 Intelligent-Tiering Variable / Desconocido Inmediata Medio Datos con patrones de acceso impredecibles. Mueve objetos automáticamente entre niveles.
S3 Standard-IA Infrecuente Inmediata Medio-bajo Backups, datos de recuperación ante desastres.
S3 One Zone-IA Infrecuente Inmediata Bajo Datos recreables, copias secundarias. Solo una AZ.
S3 Glacier Instant Rara vez Milisegundos Bajo Archivos médicos, medios de noticias con acceso ocasional.
S3 Glacier Flexible Archivo Minutos a 12 h Muy bajo Archivado a largo plazo, datos regulatorios, backups anuales.
S3 Glacier Deep Archive Archivo profundo Hasta 12 h Mínimo Retención a 7-10 años, datos raramente consultados, conformidad legal.
Funciones clave de S3
Versionado: cada carga crea una nueva versión. Permite recuperar versiones anteriores si se sobrescribe accidentalmente.
Lifecycle policies: mover objetos automáticamente entre clases según antigüedad o acceso.
Nombre de bucket globalmente único: no puede coincidir con ningún otro bucket en el mundo.
Objetos: 0 bytes a 5 TB. Datos en una región específica; no salen sin acción explícita.
03 · Amazon EBS — Tipos de volumen

SSD vs HDD · ¿Cuál para cada carga?

Tipo Tecnología Volumen de arranque Caso de uso
gp3 / gp2 — SSD uso general SSD Sí La mayoría de cargas de trabajo. Entornos dev/test. SO. Opción por defecto recomendada.
io2 / io1 — SSD IOPS provisionadas SSD Sí Apps críticas con alto IOPS sostenido. Bases de datos grandes de producción.
st1 — HDD rendimiento optimizado HDD No Streaming, Big Data, data warehouses, procesamiento de logs. Alto throughput secuencial.
sc1 — HDD en frío HDD No Datos de acceso infrecuente. Menor coste de almacenamiento en EBS.
Instance Store vs EBS
Instance Store (efímero): almacenamiento local del host físico. Muy rápido. Se pierde al detener/mover la instancia.
EBS (persistente): volumen de red. Sobrevive a reinicios. Vinculado a la AZ. Se puede hacer snapshot de EBS para backup.
04 · Sistemas de archivos administrados

EFS · FSx for Windows · FSx for Lustre

Amazon EFS

Sistema de archivos NFS compartido para Linux. Montable por múltiples instancias EC2 simultáneamente. Elástico y escalable. Dentro de VPC pero accesible vía VPN/Direct Connect.

Linux · NFS · compartido · elástico

FSx for Windows

Sistema de archivos SMB/NTFS para Windows. Integración con Active Directory. Alta disponibilidad multi-AZ. Backup diario en S3. SSD.

Windows · SMB · Active Directory

FSx for Lustre

Sistema de archivos distribuido en paralelo para HPC y ML. Latencia sub-milisegundo. Integra con S3. Para Linux. Nombre = Linux + clúster.

HPC · ML · Lustre · Linux · alto rendimiento
05 · Integración on-premises

AWS Storage Gateway — 3 tipos

File Gateway

Expone archivos SMB/NFS on-premises. Los datos se almacenan en S3. Caché local para datos recientes. Compatible con Windows.

Volume Gateway

Expone volúmenes iSCSI. Dos modos:
· Stored: datos en local, snapshots en S3.
· Cached: datos primarios en AWS, caché local frecuente.

Tape Gateway (VTL)

Biblioteca de cintas virtuales vía iSCSI. Almacenado en S3/Glacier. Para software de backup que usa cintas físicas.

06 · Backup centralizado

AWS Backup

AWS Backup
Servicio completamente administrado para centralizar y automatizar la protección de datos en todos los servicios AWS y on-premises. Configura políticas de backup, cumplimiento y supervisión desde un solo lugar. Elimina scripts personalizados y procesos manuales.
▲ Exam Tips — Puntos clave
  • S3 = objetos. EFS/FSx = archivos (compartidos). EBS = bloques (dedicado por instancia).
  • S3 Glacier Deep Archive = menor coste de S3, recuperación hasta 12h. Mejor para retención legal a largo plazo.
  • Versionado S3: actívalo para proteger contra sobrescrituras y borrados accidentales.
  • EBS solo conecta a instancias de la misma AZ.
  • Solo SSD (gp3, io2) pueden ser volúmenes de arranque. Los HDD (st1, sc1) no.
  • EFS = Linux. FSx for Windows = Windows/SMB/AD. FSx for Lustre = HPC/ML/Linux.
  • Instance Store = efímero. EBS = persistente.
  • Storage Gateway: File (S3 + caché) · Volume (iSCSI) · Tape (VTL). Todos son on-premises → AWS.
  • AWS Backup = centraliza y automatiza backups de todos los servicios AWS desde un único lugar.
