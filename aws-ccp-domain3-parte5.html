01 · Amazon VPC

Tu datacenter virtual privado en AWS

Amazon Virtual Private Cloud
Red privada virtual aislada lógicamente dentro de AWS. Servicio regional, resistente por defecto (opera en múltiples AZ). Control total sobre: rangos CIDR, subredes, tablas de enrutamiento y gateways.
Anatomía de una VPC
Internet Gateway (IGW) — punto de entrada/salida a internet público
Amazon VPC regional — CIDR: 10.0.0.0/16 (ejemplo)
Subred pública (AZ-a)
Tiene ruta hacia IGW → acceso a internet
Security Group + NACL activos
Subred privada (AZ-b)
Sin ruta directa a internet
Acceso saliente opcional vía NAT Gateway

El router de VPC (gestionado por AWS, dirección Red+1) enruta tráfico entre subredes. Las tablas de enrutamiento controlan a qué tiene acceso cada subred.

VPC por defecto vs VPC personalizada
VPC por defecto: creada automáticamente por AWS en cada región. Una por región. Ya tiene IGW, subredes en cada AZ y rutas configuradas. Lista para usar.
VPC personalizada: configurada enteramente por el cliente. Privada e aislada por defecto. Toda la configuración es responsabilidad del cliente.
02 · Componentes de red en VPC

Router · Tablas de enrutamiento · IGW · NAT · NACL · SG

Componente Nivel Función Estado (stateful)
Router VPC VPC Enruta tráfico entre subredes. Alta disponibilidad. Gestionado por AWS. Dirección Red+1. N/A
Tabla de enrutamiento Subred Define a dónde puede ir el tráfico desde la subred. Una subred → una tabla de enrutamiento. Una tabla → múltiples subredes. N/A
Internet Gateway (IGW) VPC Conecta la VPC con internet y la zona pública de AWS. Una por VPC. Resistente y escalable. Hace NAT estático (IP privada ↔ IP pública). N/A
NAT Gateway Subred pública Permite a instancias privadas iniciar tráfico saliente a internet. No permite tráfico entrante iniciado desde fuera. Gestionado por AWS. N/A
NACL Subred Firewall a nivel de subred. Filtra tráfico que cruza el límite de subred. Puede hacer Deny explícito. Stateless
Security Group ENI del recurso Firewall a nivel de recurso (EC2, RDS…). Solo reglas Allow. Deny implícito. Puede referenciar otros SGs. Stateful
03 · Conectividad a servicios AWS sin internet

VPC Endpoints — Gateway vs Interface

Permiten que instancias en subredes privadas se conecten a servicios públicos de AWS sin salir a internet, sin IGW ni NAT Gateway.

Gateway Endpoint

Se añade como ruta en la tabla de enrutamiento. Usa enrutamiento con lista de prefijos.

Servicios: S3 · DynamoDB

Interface Endpoint

Crea una ENI con IP privada en la subred. Usa DNS para redirigir el tráfico. Para todos los demás servicios AWS.

Servicios: todos salvo S3/DynamoDB (también para PrivateLink)
AWS PrivateLink
Servicio de punto de enlace de VPC que permite exponer una aplicación a otras VPCs y cuentas AWS sin interconexión de VPC ni puertas de enlace adicionales. Más seguro y escalable para arquitecturas multi-cuenta.
04 · Conectividad con entornos externos

VPC Peering · VPN · Direct Connect

Servicio Qué conecta Tránsito Característica clave
VPC Peering VPC ↔ VPC (misma o diferente cuenta/región) Red privada AWS Comunicación directa por IPs privadas. Tráfico cifrado por la red global de AWS.
AWS VPN On-premises ↔ VPC Internet pública (cifrada) Túnel cifrado sobre internet. Rápido de configurar. Latencia variable.
AWS Direct Connect On-premises ↔ AWS Fibra física dedicada No usa internet. Latencia consistente y predecible. Mayor ancho de banda. Keyword: "dedicado".
VPN vs Direct Connect
VPN: sobre internet pública, cifrado, económico, rápido de desplegar, latencia variable.
Direct Connect: fibra física privada, no pasa por internet, latencia baja y consistente, requiere tiempo de provisión (semanas/meses). Palabra clave: dedicado · privado · consistente.
05 · DNS administrado

Amazon Route 53

Amazon Route 53
Servicio DNS administrado de AWS. Resistencia global (base de datos única replicada entre regiones). Dos funciones principales:

1. Registro de dominios — comprar y gestionar nombres de dominio.
2. Hosting de zonas DNS — traducir nombres de dominio a IPs y viceversa.

Políticas de enrutamiento para alta disponibilidad:
· Failover: redirige a un recurso secundario si el primario falla.
· Ponderada: distribuye tráfico entre recursos según pesos (A/B testing, migrations).
· Basada en latencia: dirige al recurso con menor latencia para el usuario.
▲ Exam Tips — Puntos clave
  • VPC = servicio regional, opera en múltiples AZ. Una VPC por defecto por región.
  • Añadir ruta hacia IGW en la tabla de enrutamiento de una subred → esa subred pasa a ser pública.
  • NAT Gateway en subred pública permite que instancias privadas accedan a internet (solo salida).
  • NACL = stateless (reglas entrada Y salida explícitas). SG = stateful (entrada cubre salida automáticamente).
  • Solo NACL puede hacer Deny explícito. Los SGs no.
  • Gateway Endpoint → S3 y DynamoDB. Interface Endpoint → resto de servicios AWS.
  • Direct Connect = conexión física dedicada. No usa internet. Latencia consistente.
  • VPN = cifrado sobre internet pública. Más económico, más rápido de desplegar.
  • Route 53 = DNS global con resistencia global. Políticas: failover, ponderada, latencia.
  • VPC Peering: dos VPCs se comunican por IPs privadas. Puede ser cross-account y cross-region.
