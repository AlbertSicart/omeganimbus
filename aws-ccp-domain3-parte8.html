01 · Observabilidad y monitorización

CloudWatch · X-Ray · EventBridge

Amazon CloudWatch

Métricas, logs y eventos de todos los servicios AWS. Alarmas que disparan acciones automáticas cuando una métrica supera un umbral. Base del Auto Scaling de EC2.

AWS X-Ray

Tracing distribuido. Depura y analiza microservicios y apps serverless. Identifica cuellos de botella y errores en arquitecturas complejas.

Amazon EventBridge

Bus de eventos serverless. Responde casi en tiempo real a cambios en el entorno AWS. Conecta apps con eventos de servicios AWS o de terceros.

02 · Mensajería asíncrona

Amazon SQS — Colas estándar vs FIFO

Cola Estándar

  • Escalado casi ilimitado
  • Entrega at-least-once (puede duplicar)
  • Orden no garantizado (best-effort)
  • Máximo throughput
  • Acepta duplicados y desorden

Cola FIFO

  • Orden garantizado (first-in, first-out)
  • Exactly-once delivery (sin duplicados)
  • Throughput limitado (3.000 msg/s con batching)
  • Para procesos donde el orden importa (pagos, inventario)
Short Polling vs Long Polling
Short polling: responde inmediatamente aunque no haya mensajes → muchas llamadas API vacías = costoso.
Long polling (WaitTimeSeconds): espera hasta que hay mensajes o expira el tiempo → menos llamadas API. Recomendado.
03 · Notificaciones y email

Amazon SNS vs Amazon SES

Amazon SNS — Pub/Sub

  • Publicación/suscripción (fanout)
  • Suscriptores: SQS, Lambda, HTTP, email, SMS
  • Usado por CloudWatch, Auto Scaling, CloudFormation
  • Notificaciones inmediatas, configuración mínima

Amazon SES — Email

  • Simple Email Service
  • Envío masivo de email desde apps
  • Ideal para correo desde dominio personalizado
  • Marketing, transaccional, notificaciones email
Clave examen: SNS vs SES
SNS = notificaciones push a múltiples tipos de suscriptores. Sin dominio personalizado, mínima configuración.
SES = email profesional desde tu dominio, a escala. Más configuración, más control del correo.

▲ Exam Tips — Puntos clave
  • SQS = desacoplamiento, mensajería asíncrona. Keyword: "desacoplar" o "escalado independiente".
  • Cola FIFO = orden garantizado + sin duplicados. Cola estándar = mayor throughput pero puede desordenar/duplicar.
  • Long Polling (WaitTimeSeconds) = más eficiente que short polling. Reduce llamadas API vacías.
  • SNS = pub/sub, fanout a múltiples suscriptores. SES = email a escala desde dominio personalizado.
  • EventBridge = bus de eventos serverless, responde a cambios en tiempo real.
  • CloudWatch = métricas + logs + alarmas. X-Ray = tracing distribuido para microservicios.
  • WorkSpaces = escritorio virtual (VDI). AppStream 2.0 = app en streaming desde navegador, sin instalación.
  • Trusted Advisor completo disponible desde el plan Business. Basic/Developer solo tienen 7 checks.
  • IoT Greengrass = edge computing IoT (procesa datos localmente sin depender de la nube).
