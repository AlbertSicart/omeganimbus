01 · Definición

¿Qué es AWS?

Amazon Web Services
La plataforma cloud más utilizada y completa del mundo. Ofrece más de 200 servicios gestionados desde centros de datos distribuidos globalmente. Permite a empresas de todos los tamaños reducir costes, aumentar agilidad e innovar más rápido.
02 · Fundamentos

Los 5 criterios de la computación en la nube

Para que una plataforma se considere cloud computing, debe cumplir estos cinco criterios:

01
Autoservicio bajo demanda

Aprovisionas recursos (servidores, BD, redes) sin intervención humana. En AWS: consola, CLI, APIs.

02
Conectividad de red

Acceso a recursos vía consola, CLI, HTTP/S, VPN, SSH, etc.

03
Agrupamiento de recursos

AWS agrupa miles de servidores compartidos entre clientes. No conoces la ubicación exacta del hardware. AWS aplica economías de escala para trasladar el ahorro al cliente.

04
Elasticidad

Capacidad de escalar recursos hacia arriba o hacia abajo según la demanda, de forma automática.

05
Uso supervisado y facturado

Solo pagas lo que usas. No hay sobreaprovisionamiento ni infraprovisionamiento. Pay-as-you-go.

03 · Infraestructura

Infraestructura global de AWS Ver Dominio 3

Colección de agrupaciones de infraestructura conectadas por red global de alta velocidad
Permite diseñar sistemas resistentes y de alta disponibilidad. Los tres conceptos clave son:

Regiones → agrupaciones geográficas de centros de datos
Zonas de disponibilidad (AZ) → uno o más centros de datos aislados dentro de una región
Ubicaciones periféricas (Edge Locations) → puntos de presencia para distribución de contenido (CDN)
04 · Resiliencia

Alta disponibilidad vs. Tolerancia a fallos vs. Recuperación ante desastres

Concepto Definición Tiempo inactividad Coste relativo Ejemplo
Alta Disponibilidad Diseño para minimizar interrupciones y recuperarse rápido de fallos. Se acepta un tiempo de inactividad breve. Mínimo Bajo 2 servidores: uno activo, uno en espera. Conmutación automática.
Tolerancia a Fallos El sistema sigue operando aunque fallen uno o varios componentes. Cero interrupción. Ninguno Alto 2 servidores activos simultáneamente sirviendo la misma app.
Recuperación ante Desastres Plan documentado de qué hacer cuando ocurre un desastre. Se planifica con antelación. Variable (según plan) Variable Runbooks, backups cross-region, RTO/RPO definidos.
⚠ Distinción clave para el examen
Alta disponibilidad y tolerancia a fallos tratan sobre cómo diseñas el sistema para operar durante el desastre. La recuperación ante desastres trata sobre qué haces después del desastre para restaurar la operación.
05 · Escalado y Elasticidad

Escalado vertical · Escalado horizontal · Elasticidad

↑ Escalado Vertical

  • Qué es: Aumentar el tamaño de una instancia existente
  • Ejemplo: t2.micro → t2.medium
  • Ventaja: No modifica la app
  • Desventaja: Requiere reinicio (downtime breve)
  • Coste: Aumenta con el tamaño

↔ Escalado Horizontal

  • Qué es: Añadir más instancias del mismo tamaño
  • Ejemplo: 1 instancia → 3 instancias + Load Balancer
  • Ventaja: Sin downtime, más barato
  • Desventaja: Gestión de sesiones (sticky sessions)
  • Coste: Instancias pequeñas = más económico

⚡ Elasticidad

  • Qué es: Automatización + escalado horizontal
  • Objetivo: Ajustar capacidad a demanda variable
  • Servicio AWS: Auto Scaling + Launch Configurations
  • Optimiza: Rendimiento + coste + operaciones
  • Demanda: Raramente lineal → escala y desescala
▲ Exam Tips — Puntos clave
  • AWS = +200 servicios cloud. Definición core: autoservicio, red, agrupamiento, elasticidad, pay-per-use.
  • Escalado vertical = instancia más grande (redimensionar). Implica reinicio.
  • Escalado horizontal = más instancias del mismo tamaño + Load Balancer.
  • Elasticidad = automatización + escalado horizontal para seguir la demanda en tiempo real.
  • Alta disponibilidad ≠ tolerancia a fallos. HA acepta downtime breve; FT no acepta ninguno.
  • Recuperación ante desastres ≠ HA ni tolerancia a fallos. Es un plan, no un diseño de arquitectura.
  • Pay-as-you-go = solo pagas lo que usas. Elimina el sobreaprovisionamiento.
  • Economías de escala = AWS escala global → traslada ahorro al cliente.
