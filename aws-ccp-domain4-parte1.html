01 · Modelos de precios EC2

Bajo demanda · Reservadas · Spot · Savings Plans · Dedicadas

ModeloDescuento vs On-DemandCompromisoInterrupciónIdeal para
Bajo demanda (On-Demand) Sin descuento Ninguno No interrumpible Cargas impredecibles, corto plazo, apps que no toleran interrupciones.
Instancias Reservadas Hasta 72 % 1 o 3 años No interrumpible Uso constante, largo plazo, producción estable. Pagas aunque no la uses.
Instancias Spot Hasta 90 % Ninguno Puede interrumpirse Cargas tolerantes a interrupciones, procesamiento por lotes, HPC, ML.
Savings Plans Hasta 72 % 1 o 3 años No interrumpible Flexibilidad entre instancias, Lambda y ECS/Fargate con compromiso de uso ($/hora).
Hosts dedicados Variable Opcional (1-3 años) No interrumpible Licencias de software por socket/núcleo, cumplimiento normativo, arrendatario único. Pagas por el host completo.
Instancias dedicadas Variable Ninguno No interrumpible Hardware de arrendatario único sin gestionar el host físico. Pagas por instancia/hora.
Clave de examen — Spot vs On-Demand
On-Demand: keyword "no puede tolerar interrupciones", "horario flexible", "carga de trabajo a corto plazo".
Spot: keyword "precio más bajo", "tolera interrupciones", "aplicación flexible", "procesamiento por lotes".
Instancias Reservadas: keyword "uso constante", "largo plazo", "producción estable". Pagas aunque no las uses.
02 · Tipos de instancias reservadas

Standard · Convertible · Programadas

Standard (Estándar)

Mayor descuento (hasta 72 %). Menos flexibilidad. No se puede cambiar la familia de instancia. Se puede vender en el marketplace de RI.

Convertible

Menor descuento que Standard. Permite cambiar familia de instancia, SO o tenencia durante el periodo de vigencia.

Programadas

Capacidad reservada en ventanas de tiempo recurrentes predecibles (ej. cada día de 9 a 17h). Solo pagas por ese periodo.

Opciones de pago por adelantado
All Upfront: pago completo por adelantado → mayor descuento.
Partial Upfront: pago parcial + cuota mensual.
No Upfront: sin pago inicial, solo cuota mensual → menor descuento.

Facturación unificada (AWS Organizations): todas las cuentas de la organización se tratan como una sola a efectos de facturación. El beneficio de una RI comprada por cualquier cuenta aplica a toda la organización.
03 · Pilar de optimización de costes (WAF)

Prácticas clave para reducir costes

📐

Dimensionamiento correcto

Elegir el tipo y tamaño de instancia más económico que cumpla los requisitos de rendimiento. Usar AWS Trusted Advisor para identificar instancias sobredimensionadas.

⚡

Aumentar la elasticidad

Usar Auto Scaling para escalar/desescalar según demanda. Pagar solo por los recursos usados en cada momento.

💰

Modelo de precios correcto

Combinar On-Demand + Reserved + Spot según la naturaleza de cada carga de trabajo para minimizar el coste total.

💾

Optimizar el almacenamiento

Elegir la clase de almacenamiento adecuada (S3 Glacier vs Standard). Usar lifecycle policies para mover datos automáticamente a clases más baratas.

🌐

Reducir transferencia de datos

El tráfico de salida a internet y a on-premises tiene coste. Usar CloudFront y Direct Connect para reducir cargos de transferencia. Datos entre servicios en la misma región suelen ser gratuitos.

📊

Medir y supervisar

CloudWatch para métricas. Cost Explorer para tendencias. Etiquetas de asignación de costes para ver qué recursos cuestan más. Revisar periódicamente.

04 · Transferencia de datos — Costes

Cuándo hay coste y cuándo no

Tipo de tráficoCosteSolución para reducirlo
Datos de entrada a AWS (ingress) Generalmente gratis —
Tráfico entre servicios en la misma región Generalmente gratis Diseñar servicios en la misma región.
Tráfico entre AZ dentro de la misma región Pequeño cargo Minimizar tráfico cross-AZ innecesario.
Datos de salida a internet (egress) Tiene coste Usar CloudFront para reducir latencia y coste. Precios escalonados (más volumen = menos por GB).
Datos de salida a on-premises Tiene coste Usar AWS Direct Connect — el coste de la transferencia se incluye en el servicio.
Tráfico entre regiones Tiene coste Consolidar recursos en menos regiones donde sea posible.

▲ Exam Tips — Puntos clave
  • On-Demand = sin compromiso, no interrumpible. Para cargas impredecibles o corto plazo.
  • Spot = hasta 90 % de descuento, pero interrumpible. Para cargas tolerantes a fallos.
  • Instancias Reservadas Standard = mayor descuento (72 %) con mayor compromiso. Pagas aunque no las uses.
  • Convertible = menor descuento que Standard pero permite cambiar familia/SO durante la vigencia.
  • Savings Plans = similar a RI pero más flexible (aplica a Lambda y Fargate también).
  • Hosts dedicados: pagas por el host completo. Instancias dedicadas: pagas por instancia en hardware dedicado.
  • Facturación unificada (Organizations): toda la org comparte el beneficio de las RIs de cualquier cuenta miembro.
  • Egress a internet = tiene coste. Ingress a AWS = generalmente gratis.
  • Direct Connect reduce costes de transferencia de datos on-premises ↔ AWS.
