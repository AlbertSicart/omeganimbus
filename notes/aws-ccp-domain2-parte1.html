Modelo de Responsabilidad Compartida · Seguridad DE la nube vs EN la nube
01 · Definición

¿Qué es el Modelo de Responsabilidad Compartida?

Shared Responsibility Model
Marco que define qué áreas de seguridad pertenecen a AWS y cuáles son responsabilidad del cliente. La regla general es:

AWS es responsable de la seguridad DE la nube (infraestructura física, hardware, hipervisor).
El cliente es responsable de la seguridad EN la nube (datos, apps, configuración, acceso).
02 · Capas de responsabilidad

03 · Detalle de responsabilidades

AWS vs Cliente — ¿Quién hace qué?

☁ AWS — Seguridad DE la nube
  • Seguridad física de los centros de datos
  • Hardware: servidores, switches, routers
  • Infraestructura de red global
  • Hipervisores y virtualización
  • Parches del motor de BD en RDS
  • Parches del SO en Lambda, ECS (Fargate)
  • Disponibilidad y durabilidad de S3, DynamoDB
  • Certificaciones de conformidad (ISO, SOC, PCI)
👤 Cliente — Seguridad EN la nube
  • Sistema operativo en instancias EC2 (parches)
  • Configuración de security groups y NACLs
  • Gestión de identidades y accesos (IAM)
  • Cifrado de datos en reposo y en tránsito
  • Protección de datos de clientes
  • Backups y recuperación de datos propios
  • Código y configuración de las aplicaciones
  • Gestión de claves de cifrado (KMS)
04 · Clave para el examen

Responsabilidad según tipo de servicio

La responsabilidad del cliente varía según si el servicio es administrado o no. A más gestión por parte de AWS, menos responsabilidad para el cliente.

Servicio Tipo Parches SO / Motor BD Configuración de red Datos y acceso
Amazon EC2 No administrado Cliente Cliente Cliente
Amazon RDS Administrado AWS Compartido Cliente
Amazon DynamoDB Totalmente administrado AWS AWS Cliente
AWS Lambda Serverless AWS AWS Cliente
Amazon S3 Totalmente administrado AWS AWS Cliente
Regla práctica
¿Es un servicio administrado? → AWS gestiona la infraestructura subyacente, el SO y los parches del motor. El cliente solo gestiona datos, acceso y configuración de la app.

¿Es EC2 u otro servicio IaaS? → El cliente es responsable de todo lo que está dentro de la instancia: SO, parches, firewall, aplicación.
▲ Exam Tips — Puntos clave
  • AWS = seguridad DE la nube. Cliente = seguridad EN la nube. Esta distinción es la base de todo.
  • ¿Quién parchea el motor de MySQL en RDS? → AWS. ¿Y en EC2 con MySQL instalado? → El cliente.
  • ¿Quién protege los centros de datos físicos? → Siempre AWS. El cliente no tiene acceso ni control.
  • El cliente siempre es responsable de sus datos, IAM, cifrado y backups, independientemente del servicio.
  • A mayor nivel de abstracción del servicio (Lambda > RDS > EC2), menor responsabilidad del cliente sobre la infraestructura.
  • Conocer el modelo para: EC2, RDS, DynamoDB, Lambda y S3 — los más preguntados en el examen.
