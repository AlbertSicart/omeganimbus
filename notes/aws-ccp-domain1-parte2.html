01 · Definición

¿Qué es AWS Well-Architected Framework?

AWS Well-Architected Framework
Conjunto de prácticas recomendadas y estrategias de AWS para arquitectar sistemas en la nube. Ayuda a diseñar, construir y gestionar sistemas fiables, seguros, eficientes y rentables. Se estructura en 6 pilares y un conjunto de principios de diseño generales.

AWS también ofrece la Well-Architected Tool: revisa tus cargas de trabajo y las compara con las prácticas recomendadas más recientes de AWS.

02 · Base común

Principios de diseño generales

Eliminar suposiciones sobre capacidad → usar Auto Scaling
Probar sistemas a escala de producción
Automatizar la arquitectura para facilitar la experimentación
Permitir cambios evolutivos en la arquitectura
Usar datos para tomar decisiones de mejora
Mejorar mediante game days y pruebas de funcionamiento
03 · Los 6 pilares

Well-Architected Framework — Pilares y principios

01
⚙️
Excelencia Operativa
Apoyar el desarrollo y ejecutar cargas de trabajo con eficacia. Obtener información sobre operaciones y mejorar continuamente los procesos para aportar valor empresarial.
  • Operar como código
  • Cambios frecuentes, pequeños y reversibles
  • Refinar procedimientos de operaciones con frecuencia
  • Anticiparse a los errores
  • Aprender de todos los errores operativos
02
🔒
Seguridad
Proteger datos, sistemas y activos. Aprovechar las tecnologías cloud para mejorar la seguridad de la organización.
  • Implementar una base de identidad sólida
  • Mantener la trazabilidad
  • Aplicar seguridad a todas las capas
  • Automatizar prácticas recomendadas de seguridad
  • Proteger datos en tránsito y en reposo
  • Restringir el acceso a los datos
  • Prepararse para eventos de seguridad
03
🛡️
Fiabilidad
Capacidad de una carga de trabajo para llevar a cabo su función de forma correcta y uniforme cuando se espera que lo haga. Incluye gestionar y probar la carga a lo largo de todo su ciclo de vida.
  • Recuperarse automáticamente de los errores
  • Probar los procedimientos de recuperación
  • Escalar horizontalmente para aumentar disponibilidad
  • Dejar de hacer suposiciones sobre la capacidad
  • Gestionar cualquier cambio en la automatización
04
⚡
Eficiencia del Rendimiento
Utilizar los recursos de computación de forma eficiente para satisfacer los requisitos del sistema y mantener esa eficiencia a medida que la demanda y la tecnología evolucionan.
  • Democratizar las tecnologías avanzadas
  • Lograr alcance global en minutos
  • Usar arquitecturas serverless
  • Experimentar más a menudo
  • Comprender el funcionamiento del sistema
05
💰
Optimización de Costes
Ejecutar sistemas para ofrecer valor empresarial al precio más bajo posible.
  • Adoptar un modelo de consumo
  • Medir la eficiencia general
  • Dejar de gastar en tareas indiferenciadas
  • Analizar y atribuir el gasto
  • Usar servicios administrados para reducir costes
06
🌿
Sostenibilidad
Minimizar el impacto medioambiental de las cargas de trabajo en la nube, especialmente el consumo y la eficiencia energética.
  • Conocer el impacto medioambiental
  • Establecer objetivos de sostenibilidad
  • Maximizar el uso de los recursos
  • Prever y aprovechar hardware y software más eficientes
  • Usar servicios administrados
  • Reducir el impacto posterior de las cargas de trabajo
04 · Para el examen

Patrones de diseño clave a conocer

Diseño a prueba de fallos

Asumir que los componentes fallarán. Diseñar la arquitectura para que el sistema siga operando ante un fallo.

Desacoplamiento

Separar componentes para que no dependan directamente unos de otros. Principio de arquitectura orientada a servicios (SOA).

Elasticidad

Implementar escalado automático para ajustar la capacidad a la demanda. Clave para optimizar coste y rendimiento.

Paralelización

Dividir un trabajo en unidades simples y distribuirlas entre varios componentes. Ejemplo: fragmentar grandes datasets para procesamiento simultáneo.

▲ Exam Tips — Puntos clave
  • El WAF tiene 6 pilares: Excelencia Operativa, Seguridad, Fiabilidad, Eficiencia del Rendimiento, Optimización de Costes, Sostenibilidad.
  • El examen presentará escenarios prácticos: deberás identificar qué pilar o principio aplica.
  • Fiabilidad incluye recuperación automática y escalado horizontal — se solapa con HA de la Parte 1.
  • Sostenibilidad es el 6.º pilar — añadido recientemente. No confundir con optimización de costes.
  • Paralelización ≠ desacoplamiento. Desacoplar separa dependencias; paralelizar distribuye la carga de trabajo.
  • Well-Architected Tool = herramienta de AWS que evalúa tu arquitectura contra las prácticas del WAF.
  • Optimización de costes se desarrolla en profundidad en el Dominio 4.
