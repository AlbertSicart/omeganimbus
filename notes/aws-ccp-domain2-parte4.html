02 · Comparativa clave para el examen

NACL vs Security Group

Característica NACL (Network ACL) Security Group
Nivel de operación Subred (límite de subred) Recurso (ENI de EC2, RDS…)
Estado (stateful) Sin estado (stateless) — tráfico entrada y salida son flujos independientes Con estado (stateful) — si entra tráfico, la salida se permite automáticamente
Reglas de entrada y salida Deben definirse explícitamente por separado La regla de entrada cubre también la salida del mismo flujo
Denegación explícita Sí — puede crear reglas de Deny explícito No — solo Allow. Lo no permitido se deniega implícitamente
Reglas por orden Evaluadas en orden numérico ascendente. Primera coincidencia gana. Todas las reglas se evalúan. Se aplica la unión de los permisos.
Referencia a recursos AWS No — solo IPs y rangos CIDR Sí — puede referenciar otros Security Groups o a sí mismo
Cuándo usarlo Denegar tráfico explícitamente, bloquear IPs específicas a nivel de subred Control de acceso habitual a nivel de instancia o servicio
Regla crítica para el examen
Los Security Groups NO pueden hacer Deny explícito. Si necesitas bloquear una IP o rango de IPs específico, debes usar una NACL. Los SGs solo tienen reglas Allow; todo lo que no está permitido se deniega por defecto de forma implícita.
03 · Filtrado de tráfico web

AWS WAF — Cuándo elegirlo

AWS WAF vs NACL vs Security Group
NACL y SG operan a nivel de red (IPs, puertos, protocolos).
AWS WAF opera a nivel de aplicación (capa 7): filtra tráfico HTTP/S según direcciones IP, cabeceras HTTP, URLs, patrones de cuerpo de la solicitud.

Casos de uso de WAF: bloquear inyecciones SQL, XSS, bots, IPs geográficas, rate limiting por URL.

▲ Exam Tips — Puntos clave
  • NACL = subred, stateless. Debes definir reglas de entrada Y salida por separado. Puede hacer Deny explícito.
  • Security Group = recurso (ENI), stateful. Solo reglas Allow. Deny implícito. No puede hacer Deny explícito.
  • Si la pregunta pide bloquear una IP específica → NACL (SG no puede hacer Deny explícito).
  • Tráfico entre instancias dentro de la misma subred → la NACL no interviene.
  • AWS WAF = capa 7 (HTTP/S). NACL y SG = capa 3/4 (red/transporte).
  • Trusted Advisor = recomendaciones en 5 áreas (seguridad, coste, rendimiento, tolerancia, límites).
  • AWS Marketplace = software de terceros certificado para desplegar en AWS. No es un servicio AWS nativo.
  • Pen testing permitido sin aprobación previa solo en servicios específicos autorizados por AWS.
