02 · Amazon RDS en detalle

Single-AZ · Multi-AZ · Read Replica

Single-AZ

Instancia y almacenamiento en una sola AZ. Vulnerable a fallos de esa AZ.

Sin réplica

Multi-AZ

Instancia primaria + réplica en espera en otra AZ. Failover automático. Mejora disponibilidad, no rendimiento de lectura.

Replicación SÍNCRONA

Read Replica

Réplica de solo lectura. Descarga lecturas de la primaria. Mejora rendimiento y disponibilidad.

Replicación ASÍNCRONA
Palabra clave de replicación
Síncrona → Multi-AZ (alta disponibilidad, failover). Los datos se escriben en primaria y réplica al mismo tiempo.
Asíncrona → Read Replica (rendimiento de lectura). Los datos se replican con un pequeño retraso.
03 · Amazon Aurora

Diferencias clave con RDS

Aurora vs RDS
Arquitectura de clúster: 1 instancia primaria + réplicas de lectura (que también ofrecen beneficio multi-AZ).
Almacenamiento compartido en clúster: no usa almacenamiento local. Más rápido, más disponible, mejor rendimiento.
Aurora Serverless: sin aprovisionar instancias. Escala automáticamente en ACUs (Aurora Capacity Units). Puede pausarse a 0 → solo pagas almacenamiento.
Aurora Global Database: replicación multi-región en <1s. Ideal para DR y presencia global de datos en solo lectura.
04 · Bases de datos en memoria (caché)

ElastiCache vs DAX

⚡ Amazon ElastiCache

Caché en memoria administrada. Almacena resultados de lecturas frecuentes para entregarlos sin consultar la BD. También almacena estados de sesión de usuario.

Motores soportados: Redis y Memcached.

Redis · Memcached · sesiones · rendimiento lectura
🚀 Amazon DAX

Caché en memoria diseñada exclusivamente para DynamoDB. Reduce latencia de milisegundos a microsegundos para lecturas.

Ideal para cargas con muchas lecturas y sensibles a latencia. No recomendado para lecturas fuertemente consistentes.

DynamoDB · microsegundos · lectura intensiva
05 · Amazon Redshift

OLTP vs OLAP — Data Warehouse

OLTP vs OLAP
OLTP (transaccional) → RDS, Aurora, DynamoDB. Filas. Actualizar registros individuales rápidamente.
OLAP (analítico) → Redshift. Columnas. Consultas agregadas sobre grandes volúmenes de datos.

Redshift está basado en PostgreSQL pero no es para transacciones. Escala a petabytes. Arquitectura de clúster. Descarga/carga datos desde/hacia S3. Redshift Spectrum permite consultar datos directamente en S3 sin cargarlos.
06 · Migración de datos físicos — Snow Family

Snowcone · Snowball Edge · Snowmobile

Dispositivo Capacidad Compute Cuándo usarlo
AWS Snowcone Hasta 8 TB Limitado Migraciones pequeñas, ubicaciones remotas con espacio reducido.
AWS Snowball Edge TB a PB EC2 + Lambda (edge computing) Migración de datos + computación en el dispositivo. Dos versiones: Storage Optimized y Compute Optimized.
AWS Snowmobile PB a EB No Migración masiva de datacenters enteros. Camión físico.
07 · Herramientas de migración de BD

DMS · Schema Conversion Tool · DataSync

AWS DMS

Database Migration Service. Migra BD con mínimo downtime usando replicación continua. Origen activo hasta el corte final. Homogéneo o heterogéneo.

Schema Conversion Tool

Convierte esquemas entre diferentes motores de BD (ej. Oracle → Aurora PostgreSQL). Complemento de DMS para migraciones heterogéneas.

AWS DataSync

Mueve grandes volúmenes de datos online entre almacenamiento on-premises y S3, EFS o FSx. Hasta 10× más rápido que herramientas open-source. Usa internet o Direct Connect.

▲ Exam Tips — Puntos clave
  • Relacional → RDS o Aurora. NoSQL → DynamoDB. Analítico/OLAP → Redshift.
  • Multi-AZ = replicación síncrona (alta disponibilidad). Read Replica = replicación asíncrona (rendimiento lectura).
  • Aurora ≠ RDS: clúster compartido, réplicas de lectura también dan beneficio multi-AZ, más rendimiento.
  • Aurora Serverless: escala a 0, paga solo almacenamiento al pausarse. Ideal para cargas impredecibles.
  • ElastiCache = Redis o Memcached. DAX = solo para DynamoDB, microsegundos.
  • Redshift = OLAP, columnar, petabytes. No para transacciones individuales.
  • Snowcone (8TB) · Snowball Edge (TB-PB, tiene compute) · Snowmobile (PB-EB, camión físico).
  • DMS = migración con mínimo downtime. SCT = conversión de esquemas entre motores. DataSync = transferencia de archivos online.
