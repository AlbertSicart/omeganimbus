01 · Resistencia de servicios AWS

Global · Regional · Zonal — Tres niveles de resistencia

NIVEL 1
Resistencia global

Opera globalmente. Los datos se replican en múltiples regiones. Sobrevive al fallo de una región completa.

IAM · CloudFront · Amazon Route 53
Falla si: falla AWS global (prácticamente imposible)
NIVEL 2
Resistencia regional

Opera en una región. Replica datos a múltiples AZ dentro de esa región. Sobrevive al fallo de una AZ.

Amazon EFS · AWS Batch · S3 (por defecto)
Falla si: falla la región completa
NIVEL 3
Resistencia zonal

Vinculado a una AZ concreta. Si la AZ falla, el servicio también falla.

Amazon EBS · Amazon EC2 (instancia individual)
Falla si: falla su AZ específica
02 · Jerarquía de componentes

De centros de datos a infraestructura global

Estructura anidada de la infraestructura AWS
Infraestructura global AWS
Red global de alta velocidad que conecta todas las regiones. IAM, CloudFront y Route 53 operan a este nivel.
Múltiples regiones en todos los continentes
Región AWS
Zona geográfica con 2 o más AZ. Los datos en una región NO salen de ella salvo que se muevan explícitamente. Cada región es soberana y tolerante a fallos de otras regiones.
eu-west-1 (Irlanda) · us-east-1 (Virginia) · ap-southeast-1 (Singapur)
Zona de disponibilidad (AZ)
Uno o más centros de datos físicamente separados dentro de la región. Alimentación redundante, red propia. Conectadas entre sí con redes redundantes de alta velocidad.
eu-west-1a · eu-west-1b · eu-west-1c
Centro de datos
Edificio físico con servidores, SAN, switches, load balancers, firewalls, almacenamiento. Puede haber más de uno por AZ.
Hardware físico — gestionado completamente por AWS

Extensiones de región: Zonas locales (mayor cercanía geográfica a usuarios) · AWS Wavelength (latencia ultrabaja para dispositivos móviles, desplegado en infraestructura de operadoras).

03 · Ubicaciones periféricas y servicios globales

CloudFront vs Global Accelerator

Servicio Qué hace Caché Protocolo Caso de uso ideal
Amazon CloudFront CDN. Almacena en caché contenido estático y dinámico en Edge Locations. Sirve desde la ubicación más cercana al usuario. Sí — caché en edge HTTP/S Distribución de contenido web, imágenes, vídeos, APIs. Reducir latencia de carga de contenido.
AWS Global Accelerator Enruta tráfico a través de la red global AWS desde las Edge Locations hasta las aplicaciones en las regiones. Sin caché. No hay caché TCP · UDP Apps que necesitan IP estática, failover regional rápido, gaming, VoIP, IoT.
Puntos en común CloudFront y Global Accelerator
Ambos usan la red global de AWS y las Edge Locations. Ambos integran protección AWS Shield contra DDoS. La diferencia clave: CloudFront almacena en caché · Global Accelerator solo enruta.
04 · Cómo elegir región

Factores para seleccionar una región AWS

⚖️

Conformidad y soberanía de datos

Los datos no salen de la región sin acción explícita. Leyes locales (GDPR, etc.) pueden exigir regiones específicas.

⚡

Latencia

Elegir la región geográficamente más cercana a los usuarios finales reduce la latencia.

🛠️

Disponibilidad de servicios

No todos los servicios AWS están disponibles en todas las regiones. Verificar antes de diseñar.

💰

Coste

Los precios de los servicios varían entre regiones. Una misma instancia puede tener distinto coste en us-east-1 vs eu-west-1.

05 · Modelos de servicio cloud

IaaS · PaaS · SaaS · DBaaS

IaaS — Infraestructura como Servicio

El proveedor gestiona hardware y virtualización. El cliente gestiona SO, runtime y apps.

Amazon EC2, Amazon VPC, Amazon EBS

PaaS — Plataforma como Servicio

El proveedor gestiona infraestructura + SO + runtime. El cliente solo despliega código.

AWS Elastic Beanstalk, AWS Lambda

SaaS — Software como Servicio

El proveedor gestiona todo. El cliente solo usa la aplicación.

Amazon Chime, Amazon WorkMail, Salesforce

DBaaS — Base de datos como Servicio

Base de datos totalmente administrada. El proveedor gestiona el motor, parches, backups y HA.

Amazon RDS, DynamoDB, Aurora
▲ Exam Tips — Puntos clave
  • 3 niveles de resistencia: Global (IAM, CloudFront, Route 53) · Regional (EFS, S3) · Zonal (EBS, EC2 individual).
  • Región = 2+ AZ. AZ = 1+ centros de datos. Los datos no salen de la región sin acción explícita.
  • CloudFront = CDN con caché en edge, HTTP/S. Global Accelerator = enrutamiento TCP/UDP sin caché, IP estática.
  • Ambos (CloudFront y Global Accelerator) usan Edge Locations y tienen Shield integrado.
  • Para elegir región: conformidad → latencia → disponibilidad de servicios → coste. En ese orden de prioridad.
  • Wavelength = latencia ultrabaja para móviles (desplegado en operadoras). Zonas locales = extensión de región geográficamente cercana.
  • AWS Outposts = infraestructura AWS en las instalaciones del cliente (nube privada).
  • IaaS (EC2) → mayor control. PaaS (Beanstalk) → solo código. SaaS → solo uso. DBaaS → BD administrada.
