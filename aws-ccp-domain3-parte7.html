01 · Tres niveles de servicios ML en AWS

IA · ML · Frameworks — De lo fácil a lo profundo

Nivel 1 — Servicios de IA
Sin conocimientos de ML
Servicios completamente administrados accesibles vía API. Modelos pre-entrenados. No requieren conocimientos de ML.
Translate · Polly · Lex · Rekognition · Comprehend · Textract · Macie
Nivel 2 — Servicios de ML
Para científicos de datos
Servicios administrados para crear, entrenar y desplegar modelos propios. Requieren conocimientos de ML.
Amazon SageMaker · Amazon CodeWhisperer
Nivel 3 — Frameworks e infraestructura
Para expertos en ML
Marcos open-source sobre infraestructura optimizada. Control total del entorno ML.
TensorFlow · PyTorch · Apache MXNet · Deep Learning AMIs · Instancias P3/P3dn EC2

03 · Servicios de análisis

Athena · Macie · Kinesis · Glue · QuickSight · EMR · Redshift

Servicio Qué hace Serverless Keyword / Caso de uso
Amazon Athena Consulta SQL interactiva sobre datos en S3. Sin ETL previo. Paga por consulta/TB escaneado. Sí "consultar S3 con SQL" · logs · reportes de costes · sin aprovisionar
Amazon Macie Detecta, clasifica y protege datos sensibles (PII) en S3 usando ML. Alertas y dashboards. Sí "PII" · información sensible · protección de datos en S3
Amazon Kinesis Ingesta y procesamiento de datos en streaming a cualquier escala: vídeo, audio, logs, IoT, clickstreams. Sí "tiempo real" · streaming · IoT · millones de eventos/s
AWS Glue Servicio ETL sin servidor. Descubre, prepara, mueve e integra datos de múltiples fuentes para análisis y ML. Sí "ETL" · catálogo de datos · preparación de datos · data lake
Amazon QuickSight Servicio de BI completamente administrado. Dashboards e informes interactivos con insights de ML para toda la organización. Sí "BI" · dashboards · visualización · informes empresariales
Amazon EMR Plataforma big data para procesar grandes cantidades de datos. Usa Hadoop, Spark sobre EC2 y S3. No "Apache Spark/Hadoop" · big data · transformaciones · ML a escala
Amazon Redshift Data warehouse columnar a escala de petabytes. Para OLAP. Redshift Spectrum consulta S3 sin cargar datos. No (clúster) "OLAP" · "data warehouse" · "petabytes" · analítico
Athena vs Redshift Spectrum vs Redshift
Athena: serverless, SQL directo sobre S3, sin clúster, paga por consulta. Para consultas ad-hoc rápidas.
Redshift Spectrum: necesita un clúster de Redshift pero consulta S3 sin cargar datos. Más potente que Athena para cargas analíticas pesadas.
Redshift: clúster completo, datos cargados dentro, máximo rendimiento analítico OLAP.
▲ Exam Tips — Puntos clave
  • Rekognition = visión artificial (imágenes/vídeo). No confundir con SageMaker (que entrena modelos).
  • Macie = detección de PII en S3 con ML. Regla mnemotécnica: Macie tiene una I, PII también.
  • Kinesis = datos en streaming en tiempo real. Palabra clave imprescindible.
  • Athena = SQL serverless sobre S3. Sin ETL, sin clúster. Paga por TB escaneado.
  • EMR = Apache Hadoop/Spark sobre EC2. Para big data y transformaciones masivas. No serverless.
  • QuickSight = BI y dashboards. No es una BD ni un motor de análisis.
  • AWS Glue = ETL serverless. Preparación e integración de datos para Athena, Redshift o EMR.
  • Nivel IA (Rekognition, Polly…) = sin conocimientos ML. Nivel ML (SageMaker) = para científicos de datos.
