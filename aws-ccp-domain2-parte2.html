01 · Conformidad

Dónde encontrar información de conformidad en AWS

Principio clave
No memorizar qué servicios cumplen qué normas. Saber dónde buscar la información y que los requisitos de conformidad varían de un servicio a otro.
📄 AWS Artifact
Acceso bajo demanda a documentos e informes de seguridad y conformidad de AWS.
Úsalo cuando necesites verificar si un servicio (ej. DynamoDB) cumple una norma (ej. GDPR, PCI-DSS, ISO 27001).
conformidad · auditoría · informes
02 · Servicios de seguridad

Protección de recursos en AWS

🔍 Amazon GuardDuty
Servicio de detección de amenazas que supervisa actividad malintencionada y comportamiento no autorizado.
Monitorización continua e inteligente de la cuenta y la red.
detección de amenazas · comportamiento anómalo
🧱 AWS WAF
Web Application Firewall. Protege aplicaciones web de vulnerabilidades comunes (SQLi, XSS…).
Cuando necesitas proteger una app web de ataques en la capa 7 (HTTP).
firewall · aplicación web · capa 7
🛡️ AWS Shield
Servicio administrado de protección contra ataques DDoS para aplicaciones en AWS.
Protección automática (Standard) o avanzada con soporte 24/7 (Advanced) contra DDoS.
DDoS · protección · disponibilidad
🔬 Amazon Inspector
Análisis automatizado de vulnerabilidades de seguridad en instancias EC2, contenedores y funciones Lambda.
Escaneo de vulnerabilidades y desviaciones de mejores prácticas.
vulnerabilidades · escaneo · EC2
🗂️ AWS Security Hub
Vista centralizada de alertas y estado de seguridad de todos los servicios AWS y herramientas de terceros.
Panel único de seguridad para toda la cuenta / organización.
centralización · alertas · postura de seguridad
03 · Cifrado

Datos en tránsito vs datos en reposo

🔄 Cifrado en tránsito

Datos que se mueven entre dos puntos. Se cifran durante la transmisión para evitar interceptación.

Ejemplos: HTTPS/TLS entre cliente y servidor, VPN, SSL entre servicios AWS.

💾 Cifrado en reposo

Datos almacenados. Se cifran en disco para protegerlos aunque el soporte físico sea comprometido.

Ejemplos: S3 SSE, RDS encryption, EBS encrypted volumes, DynamoDB encryption.
Responsabilidad del cifrado
Quién activa el cifrado depende del servicio. Para servicios administrados como S3 o RDS, AWS puede cifrarlo por defecto o el cliente lo habilita en la configuración. En EC2 + EBS, el cliente es responsable de activar el cifrado. Referencia: Modelo de Responsabilidad Compartida (D2·P1).
04 · Registro y auditoría — No confundir

CloudWatch vs CloudTrail vs AWS Config

Servicio ¿Qué hace? Tipo de dato Caso de uso típico
Amazon CloudWatch Supervisión y recopilación de datos operativos y métricas de rendimiento. Métricas · logs · alarmas CPU al 90%, latencia alta, alertas de rendimiento de la app.
AWS CloudTrail Registro de eventos de API: quién hizo qué, cuándo y desde dónde en la cuenta AWS. Eventos de API · acciones de usuario ¿Quién eliminó esa instancia EC2? ¿Quién modificó ese security group?
AWS Config Inventario de configuraciones actuales y auditoría de recursos para verificar conformidad. Estado de configuración · cambios ¿Mis S3 buckets tienen cifrado activado? ¿Ha cambiado la configuración de algún recurso?
AWS Audit Manager Automatización de la recopilación de evidencias para auditorías de conformidad. Evidencias · informes de conformidad Preparar auditorías SOC 2, PCI-DSS, HIPAA de forma continua.
Pregunta de examen frecuente
"¿Qué servicio usarías para identificar al usuario IAM que eliminó una instancia EC2?"
→ AWS CloudTrail. Registra todas las llamadas a la API con identidad del usuario, hora y origen. CloudWatch mide rendimiento; Config audita configuraciones. CloudTrail es la fuente de verdad para acciones en la cuenta.
05 · Principio de seguridad

Acceso con mínimo privilegio

Least Privilege Access
Conceder únicamente los permisos estrictamente necesarios para realizar una tarea. Nada más. No todos los usuarios de una cuenta AWS deben tener el mismo nivel de acceso. Este principio se implementa mediante IAM (próxima lección).
▲ Exam Tips — Puntos clave
  • AWS Artifact = dónde encontrar informes y documentos de conformidad. No es un servicio de seguridad activo.
  • GuardDuty = detección de amenazas y comportamiento anómalo. WAF = firewall de apps web (capa 7). Shield = protección DDoS.
  • CloudTrail = quién hizo qué en la cuenta (API calls). La respuesta para preguntas de auditoría de acciones de usuario.
  • CloudWatch = métricas y rendimiento operativo. No registra quién hizo qué.
  • AWS Config = estado y cambios de configuración de recursos. Conformidad de configuraciones.
  • Cifrado en tránsito (datos moviéndose) ≠ cifrado en reposo (datos almacenados). Conocer la diferencia.
  • Least privilege = dar solo el acceso mínimo necesario. Implementado con IAM.
  • Los requisitos de conformidad varían por servicio. No hay una respuesta única para todos.
